# シンプルメモ帳＆ToDoリスト 仕様書

このドキュメントは Simplist シリーズ第 6 作「シンプルメモ帳＆ToDoリスト」の正本仕様書。

- ビジュアルデザインは別書 `DESIGN_SPEC.md` を真とする
- シリーズ共通規約は `morphyca-app/simple_alarm/SPEC.md` および workspace memory `simplist_series.md` を参照
- 不具合調査の入り口は `CLAUDE.md`（Phase 1 着手後に作成）

---

## 0. 開発開始時に最初にやるべきこと

1. 本書 (`SPEC.md`) と `DESIGN_SPEC.md` を読む
2. AdBlock の `app/` をシリーズ canonical template として参照（コード構造・BillingManager 等）
3. **配色・テーマ周りは本アプリ専用に新規作成**：既存シリーズの `colors.xml` / `themes.xml` / `styles_extras.xml` をそのままコピーするとトーンが崩れる（§0 参照）
4. workspace memory: `simple_memo_decisions.md` / `feedback_corner_radius.md`（本アプリは例外扱い）/ `feedback_settings_icon.md` / `feedback_material_min_height.md` を確認
5. Phase 1 着手時は本書 §22 のタスクリストを上から順に

## 1. アプリ概要

| 項目 | 値 |
|---|---|
| アプリ名 (日) | シンプルメモ帳＆ToDoリスト |
| アプリ名 (英) | Simple Memo |
| パッケージ | `jp.simplist.memo` |
| プラットフォーム | Android (minSdk 26 / targetSdk 35) |
| シリーズ | Simplist (6 作目) |
| カテゴリ | 仕事効率化 |
| サイト slug | `simple-memo` |
| Privacy URL | `https://morphyca.com/products/simple-memo/privacy/` |

### コンセプト
- 日本のメモ帳アプリ市場の不満点（ウィジェット不具合・データ消失・広告過多・機能のミスマッチ）を地味に潰す
- 「広告クリーン × ウィジェット安定 × ローカル保存 × 必要十分な機能」で勝負する買い切り型シンプルメモ帳
- テキストメモとチェックリストの 2 形態を 1 アプリに統合
- 完全ローカル保存（クラウド同期なし、機種変更時の手動エクスポートで対応）

### 想定ターゲット
- メモアプリの広告に疲れている層
- 機種変更時のデータ消失に懲りた層
- Google Keep が重い／同期に不信感を持っている層
- ウィジェットで買い物リストをホーム画面に貼りたい層

## 2. シリーズ統一事項（実装フレームワーク）

### 2.1 言語・フレームワーク
- Kotlin + Material Components (View system, **Compose 不使用**)
- DI なし、ViewBinding 有効、軽量 Repository 直書き
- 親テーマ: `Theme.Material3.DayNight.NoActionBar` + 本アプリ専用オーバーライド
- 親 Activity: `AppCompatActivity`、Toolbar は手書き LinearLayout
- minSdk 26 / compileSdk 35 / targetSdk 35

### 2.2 シリーズ共通フレームワークから継承するもの
- 設定画面: 手書き LinearLayout + HairLine + chevron（PreferenceFragmentCompat 不使用）
- 設定項目順序ルール（共通項目 1〜11 固定、本アプリ固有項目を 3〜10 に挿入）
- フォント: Inter のみ
- ログ: `if (BuildConfig.DEBUG)` で全ガード
- Play 申請: `targetSdk 35`、`trialBypass=false`、Data Safety フォーム整合
- AdBlock の `app/` を canonical template

### 2.3 シリーズ共通から逸脱するもの（本アプリの特殊事情）
- **配色**: 既存シリーズの `paper #F8F8F6` / `ink` / `graphite` / `hair` パレットは継承しない。本アプリ独自のクリーム＋パステル基調を新規定義。
- **角丸**: シリーズ規約「全要素 0dp 直角」は本アプリでは適用外。リファレンスが明示的に丸めを示しているため、規約の例外条件を満たす（DESIGN_SPEC §5 参照）
- **アクセント**: 既存シリーズの `accent_calc` (olive sage) は流用しない。本アプリ独自のセージグリーンを新規定義

### 2.4 設定画面の右端アイコン
- `ic_menu` (3 本線ハンバーガー) を使用。**歯車アイコンは使わない**（シリーズ共通規約）

## 4. 画面構成

### 4.1 メイン画面（メモ一覧）
- ツールバー: ハンバーガー (≡) / タイトル / **並び替え (⇅)** / 検索 / その他メニュー (⋮)
- タグフィルタ行: 横スクロールチップ（「すべて」「未分類」＋ユーザー定義タグ＋「+追加」）
- メモ一覧: カード形式。各カードに以下を表示
  - メモ種別アイコン（📄テキスト / ☑リスト）
  - メイン表示: タイトル優先、なければ本文 / 最初の項目の冒頭 1 行
  - 優先度: 右端に★（0〜3 個、0 は非表示）
  - 背景色: メモごとに設定された色
- FAB: 「メモ作成」と「リスト作成」の 2 ボタン
- 一覧画面では更新日時・タグチップ・ゴミ箱アイコンは表示しない（カードをスッキリさせる）
- 並び順: ①手動並び替え（sortOrder 設定済）→ ②★順（設定で有効化時のみ）→ ③更新日順（updatedAt 降順）

#### 操作
- タップ → 編集画面へ遷移 (通常モード) / 選択 toggle (複数選択モード)
- 長押し → **複数選択モード**を開始 (そのメモが選択状態になる)。複数選択中はツールバーが
  「✕ 件数 タグ 保護 削除 ⋮」に切り替わり、削除 / タグ一括変更 / 保護一括トグル / 複製 /
  すべて選択が可能。
- タグフィルタチップタップ → そのタグでフィルタリング

> 旧仕様: 長押しでコンテキストメニュー (削除/複製/タグ変更/保護トグル) を表示していたが、
> 単発操作のみで複数まとめての操作ができなかったため 2026-05 に複数選択モードに統合。

#### 並び替えモード（ツールバーの ⇅ アイコンで開始）
- ツールバーの ⇅ アイコンをタップ → 並び替えモードへ移行
- 並び替えモード中の表示変化:
  - ツールバー: ⇅ アイコンが「完了」テキストボタンに変化（タップで通常モードに戻る）
  - 各メモカードの右端にドラッグハンドル（≡）が表示される
  - FAB は非表示（誤タップ防止）
  - タグフィルタ行は非表示（並び替え対象を全件にするため、フィルタリングは解除）
- 並び替え操作:
  - カードを長押し → ドラッグで上下に並び替え（`ItemTouchHelper`）
  - リリースで `sortOrder` を更新
- 戻る:
  - 「完了」ボタンまたは Android の戻るボタンで通常モードへ
  - 「すべて更新日順に戻す」メニューも提供（⋮ から、`sortOrder` を全件 null にリセット）

> **設計判断**: 並び替えモードは「明示的に切り替える」UI とし、通常時はカードをスッキリさせる。ドラッグハンドルが常時見える方式は採らない（彩度の高い 18 色カード上にハンドルが乗ると視覚ノイズになるため）。

### 4.2 メモ編集画面（テキスト）
- ツールバー: 戻る / 色選択 / 優先度 / タグ / ゴミ箱
- タイトル入力欄（任意）
- 本文入力欄
- 補助ツールバー（画面下）: 共有 / 保護
- 自動保存: 入力変更時に都度保存（debounce 500ms）
- 自動リンク: 表示モードで電話番号・URL・メールを Linkify

### 4.3 リスト編集画面（チェックリスト）
- ツールバー: 戻る / 色選択 / 優先度 / タグ / ゴミ箱
- タイトル入力欄（任意）
- 各項目: チェックボックス＋テキスト＋ドラッグハンドル
- 「+項目を追加」で新規行追加
- 改行で新規項目、空行でバックスペースすると行削除
- チェック済みは下段に自動移動（または視覚的にグレーアウト）
- 「完了済みを削除」で一括削除

### 4.4 検索画面
- インクリメンタルサーチ（入力毎にリアルタイム更新）
- 検索対象: タイトル、本文、チェックリスト項目テキスト
- 単純な部分一致（大文字小文字区別なし、ひらがなカタカナ区別あり）
- スペース区切りで AND 検索（V1 では単一語のみ。V2 で拡張可）
- ヒット箇所スニペット: 前後 30 文字を抽出、検索ワードを太字＋アクセントカラー（カード色追従）
- **検索結果カードには更新日時を表示**（メイン一覧との差分。デザイン画像で確認済み）

#### IME 対応
- 日本語 IME のコンポーズ中（未確定文字列）でも検索が走るが、コンポーズ確定の文字列を優先的に対象とする実装にする（「お」と打ち始めた瞬間に「お茶」「おかし」が大量にヒットする問題の回避）

## 5. ウィジェット仕様

### 5.1 基本仕様
- デフォルトサイズ: 4×2 セル
- リサイズ: 縦横自由（`resizeMode="horizontal|vertical"`）
- コンテンツ: 1 つの個別メモを表示（メモ一覧型ではない）
- 設定: 配置時に表示するメモを選択

### 5.2 テキストメモウィジェット
- 表示: タイトル＋本文（サイズに応じて表示行数を調整）
- 操作: タップ → アプリを開いて該当メモの編集画面

### 5.3 チェックリストウィジェット
- 表示: タイトル＋項目リスト（チェックボックス付き）
- 操作:
  - 各項目のチェックボックス → ウィジェット上で直接 ON/OFF
  - タイトル領域タップ → アプリを開いて該当リストの編集画面
- サイズ別表示量の目安:
  - 2×1: タイトルのみ／タイトル＋1 項目
  - 4×2: タイトル＋本文 3 行／タイトル＋4 項目
  - 4×4: タイトル＋本文 10 行／タイトル＋10 項目

### 5.4 ロック画面通知（チェックリスト用）
- ウィジェットに設定したチェックリストを通知としても常時表示可能
- 通知 ON/OFF はウィジェット単位
- 通知タップ → アプリを開いて該当リストを表示
- ロック画面上での直接チェック ON/OFF は不可（Android の通知仕様上できない）

## 6. 検索機能

§4.4 と重複するため詳細はそちらを参照。実装上の注意点:

## 7. タグ機能

- 最大数: 10 個
- メモあたり: 1 タグまで（複数タグ不可）
- プリセット: 仕事 / プライベート / 買い物（ユーザーによる削除・編集可）

### 7.2 操作

- メイン画面のタグフィルタ行でタグ別絞り込み

## 8. テンプレート機能

### 8.1 仕様
- プリセット: 5 つ
- ユーザー追加: 上限なし
- 編集: プリセット含めすべて編集・削除可能

### 8.2 利用フロー
- ＋ボタン（メモ／リスト）をタップ → 新規作成（テンプレートなし）
- ＋ボタンを **長押し** → テンプレート選択ダイアログ → 選択して新規作成
- 設定画面 →「テンプレート管理」で追加・編集・削除

> 長押しでテンプレート呼び出しにすることで、通常タップは「すぐ書く」フローを最短化する。

## 9. バックアップ・復元

### 9.1 自動バックアップ
- 毎日 1 回、端末アイドル + ストレージ十分のタイミングで自動実行
- 実装: `WorkManager` (PeriodicWorkRequest, 24h 周期, BackupWorker)
- 保存先: ユーザーが SAF Tree URI で選択したフォルダ（永続化 URI 権限）
- 形式: JSON (ファイル名 simple_memo_yyyyMMdd_HHmm.json、同名既存は上書き)
- 自動バックアップ ON にはフォルダ事前設定が必須 (UI 側でガード)

### 9.2 手動エクスポート
- 設定画面 →「バックアップ・復元」→「エクスポート」
- 保存先: ユーザー選択のフォルダ（SAF: `ACTION_CREATE_DOCUMENT`）
- 形式 2 種:
  - **テキスト形式 (.txt)**: 人が読める形式（推奨）
  - **JSON 形式 (.json)**: 機種変更時の完全復元用

### 9.3 復元
- 設定画面 →「バックアップ・復元」→「インポート」
- ファイル選択 → 確認ダイアログ → インポート実行
- インポート方式: 「上書き」または「マージ」を選択

### 9.4 機種変更時の手順（ユーザー向け）
1. 旧機種で JSON エクスポート
2. ファイルを Google ドライブ／メールなどで新機種へ転送
3. 新機種でインポート

> クラウド同期は提供しないが、機種変更時にデータが消えないよう手動エクスポート手段を必ず提供する。

## 10. ゴミ箱機能

- 保持期間: **固定 7 日間**
- 保持期間後: 自動的に完全削除（毎日のバックアップタスクと同時に実行）
- メモ保護ロック ON のメモを削除しようとすると確認ダイアログを表示

### 操作
- 一覧画面でカードを長押し →「削除」→ ゴミ箱へ移動
- 編集画面のゴミ箱アイコン → ゴミ箱へ移動
- 設定画面 →「ゴミ箱」→ 復元または完全削除

## 11. メモ保護ロック（誤削除防止）

- 各メモに「保護ロック」フラグを設定可能（編集画面の鍵アイコン）
- 保護中は **削除動作自体をブロック**：
  - 長押しメニュー →「削除」、編集画面のゴミ箱アイコン、いずれを押しても
    「このメモは保護されています。削除するには先に保護を解除してください」案内ダイアログ（OK のみ）が出て、削除フローには進まない
  - 削除するには先に編集画面で保護を解除する必要がある
- メイン画面のメモカードに小さな鍵アイコンを表示（保護中であることが一目で分かる）
- ピン留めの代替機能ではなく、**誤削除防止 + 重要メモの明示**が目的

> **設計判断**: 当初は「保護ONなら確認文言を強める」だけの実装だったが、確認 → OK で削除可能だと「保護」と感じられないという実利用フィードバックを受けて、2026-05 に「実際にブロックする」方式に変更した。

## 12. プライバシーロック（生体認証）

### 12.1 仕様
- アプリ全体に対するロック（V1 は個別メモロックなし）
- `BiometricPrompt` API
- 生体認証（指紋・顔）が使えない端末では端末 PIN/パスワードへ自動フォールバック
- 設定画面 →「プライバシーロック」で ON/OFF 切替

### 12.2 動作
- アプリ起動時 / バックグラウンド復帰時に認証要求
- 認証成功後はアプリ終了まで再認証なし
- 認証失敗時はアプリ使用不可

### 12.3 V2 以降の検討事項
- 個別メモロック（特定メモのみロック）
- 認証要求のタイミング設定（即時／1 分後／5 分後 等）

---

## 13. 自動リンク

- Android 標準の `Linkify` API を使用
- 表示モード（編集中ではない時）に以下を自動でリンク化:
  - 電話番号 → タップで電話発信ダイアログ
  - URL → タップでブラウザ起動
  - メールアドレス → タップでメールアプリ起動
- 編集中はリンク化しない（誤タップ防止）

## 14. カラー（くすみ系18色）

### 14.1 仕様
- メモごとに背景色を選択可能
- パレット: 18 色（くすみ系・パステル調）＋ デフォルト（背景と同色＝色なし状態）
- 詳細な色コード・配置は `DESIGN_SPEC.md` §3 を真とする

### 14.2 タグ色との関係
- タグも同じ 18 色 ID 空間から色を持つ
- レンダリング差分: メモカード = 彩度そのまま、タグチップ = 彩度を下げた表示

## 15. 課金・トライアル

### 15.1 課金モデル
- **14 日間の無料トライアル**
- トライアル後: **買い切り 250 円**
- サブスクなし、広告なし

### 15.2 IAP 商品
- 商品 ID: `permanent_unlock`（シリーズ共通命名規則）
- 価格: 250 円（為替・地域別価格は Play Console 側で管理）
- 種別: 非消費型

### 15.3 トライアル中
- すべての機能が利用可能
- 設定画面に残り日数を表示
- 残り 3 日になったらトップに通知バナーを表示

### 15.4 トライアル後（未購入）

利用可能（重要）:
- ✅ 既存メモの **閲覧**
- ✅ 既存メモの **共有**（ACTION_SEND）
- ✅ ウィジェットの **チェック操作**（買い物中に困らないため）
- ✅ 検索

利用不可:
- ❌ 既存メモの **編集**
- ❌ 新規メモ・リストの **作成**
- ❌ テンプレートの追加・編集
- ❌ タグの追加・編集
- ❌ バックアップ・復元

> ウィジェットのチェック操作だけは制限しないのが本仕様の重要ポイント。買い物中にいきなり使えなくなるとレビュー炎上要因。

### 15.5 購入後
- 全機能解放
- 復元機能あり（設定 →「購入を復元」）

### 15.6 anti-reset 三層防御（AdBlock からポート）
- `SharedPreferences` (`trial_state`) を `backup_rules.xml` に登録（Auto Backup 対象）
- SSAID ハッシュで「別端末からのリストア」を検出
- `PackageInfo.firstInstallTime` を下限として使う
- salt = `"simplist-memo-v1"`、`TRIAL_DURATION_MS` = 14 日

### 15.7 trialBypass フラグ
- `gradle.properties` で `trialBypass=true`（debug 既定 ON）
- リリース時は `trialBypass=false` 必須

## 16. 共有機能

- 編集画面の共有ボタン → `Intent.ACTION_SEND`
- 共有テキスト:
  - テキストメモ: `タイトル + 本文`
  - チェックリスト: `タイトル + 各項目（チェック状態付き、例: ☑ 卵 / □ 牛乳）`
- 共有先: ユーザーがインストール済みのアプリ全て
